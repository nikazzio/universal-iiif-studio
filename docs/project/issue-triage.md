# Issue Triage

This page summarizes the repository governance rules for issues and pull requests.

## Summary

Every issue and PR should carry:

- one `type:*` label;
- one `priority:*` label;
- one `area:*` label;
- one semver label.

## Execution Model

- `P0` and `P1` belong in the active or immediate lane;
- `P2` belongs in the next planned lane;
- `P3` belongs in backlog or roadmap space.

See [Issue Triage Policy](../ISSUE_TRIAGE_POLICY.md) for the full label and milestone model.
